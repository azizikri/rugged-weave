# Conventional Commits & Release Automation

## Sources
- Conventional Commits v1.0.0 specification defining commit message structure and automation benefits. [Link](https://www.conventionalcommits.org/en/v1.0.0/)

## Notes
- Format commits as `<type>[optional scope]: <description>` with types like `feat` for features and `fix` for bug fixes; breaking changes use `!` or a `BREAKING CHANGE` footer. [Link](https://www.conventionalcommits.org/en/v1.0.0/)
- Following the spec enables automated changelog generation, semantic versioning, and clearer communication for contributors. [Link](https://www.conventionalcommits.org/en/v1.0.0/)
