# Documentation Best Practices

## Sources
- Atlassian Loom blog: nine best practices for crafting effective software documentation. [Link](https://www.atlassian.com/blog/loom/software-documentation-best-practices)
- GitHub Docs: guidance on README purpose, typical sections, and repository presentation. [Link](https://docs.github.com/en/repositories/managing-your-repositorys-settings-and-features/customizing-your-repository/about-readmes)
- Best README Template providing structured sections for project overviews, setup, usage, roadmap, and acknowledgements. [Link](https://github.com/othneildrew/Best-README-Template)

## Notes
- Tailor docs for the target audience, keep navigation clear, version docs, enforce a style guide, and review updates regularly to maintain accuracy. [Link](https://www.atlassian.com/blog/loom/software-documentation-best-practices)
- README files should explain the project purpose, usage, setup, contributing guidance, and related documentation, acting as the first touchpoint for contributors. [Link](https://docs.github.com/en/repositories/managing-your-repositorys-settings-and-features/customizing-your-repository/about-readmes)
- Use templates with sections for about, prerequisites, installation, usage, contributing, license, and acknowledgments to provide consistent onboarding. [Link](https://github.com/othneildrew/Best-README-Template)
