# Changelog Policy

## Sources
- Keep a Changelog 1.1.0 outlines human-readable changelog guidelines and alignment with Semantic Versioning. [Link](https://keepachangelog.com/en/1.1.0/)

## Notes
- Maintain a single `CHANGELOG.md` documenting notable changes with sections like Unreleased, Added, Changed, Fixed, and Removed. [Link](https://keepachangelog.com/en/1.1.0/)
- Pair changelog entries with semantic versioning so releases communicate scope clearly to users. [Link](https://keepachangelog.com/en/1.1.0/)
